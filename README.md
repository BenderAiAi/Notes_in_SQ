# NoteFlow

Локальное веб-приложение для проверки ежедневных Excel-отчётов по сделкам с деривативными нотами и формирования файла `Note_trades_sq_DDMMYYYY.xlsx` для SQ.

## Возможности

- выбирает актуальный отчёт по значению `Дата сделки` внутри файла;
- индексирует файлы и повторно читает только новые или изменённые `.xlsx`;
- при нескольких файлах за одну дату предлагает выбрать нужный;
- предупреждает, если последняя доступная дата не совпадает с сегодняшней;
- показывает число строк, купленные и проданные сделки, объёмы `Кол-во` и уникальные ISIN;
- объясняет ошибки с указанием строки и проблемного значения;
- хранит и редактирует справочник нот в локальной базе SQLite;
- импортирует и экспортирует справочник в Excel;
- формирует единственный лист `note_trades` с заданным порядком столбцов.

Все данные остаются на рабочем компьютере. Сервис слушает только `127.0.0.1`.

## Запуск на Windows

1. Убедиться, что Python доступен командой `python`.
2. При необходимости установить зависимости из утверждённого набора:

   ```bat
   pip install -r requirements.txt
   ```

3. Дважды нажать `start.bat`.
4. Интерфейс автоматически откроется по адресу [http://127.0.0.1:17843](http://127.0.0.1:17843).

Остановить приложение можно сочетанием `Ctrl+C` в окне консоли.

## Первый запуск

1. На вкладке «Отчёт» выбрать папку с исходными отчётами и папку для готовых файлов.
2. На вкладке «Справочник» добавить записи вручную или импортировать `.xlsx` с колонками:

   `Note | ISIN | Note name SQ | Portfolio | Subportfolio | Subaccount`

3. Вернуться на вкладку «Отчёт» и нажать «Проверить и подготовить».
4. Исправить показанные ошибки или сформировать файл.

Поле `Note` справочника является справочным. В выходной столбец `Note` попадает значение `Note name SQ`.

## Выходной файл

- имя: `Note_trades_sq_DDMMYYYY.xlsx`;
- лист: `note_trades`;
- столбцы:

  `ID | Note | Buy | Amount | Price | Portfolio | Subportfolio | Subaccount | Traded Date Time | Settlement Date`

В оба поля даты и времени переносится полное время сделки, включая секунды.

## Демонстрационные данные

Реальные данные не входят в репозиторий. Для локальной проверки можно создать полностью синтетический пример:

```bash
python scripts/create_demo_data.py
```

После выполнения появятся папки `demo/reports`, `demo/ready` и демонстрационный справочник.
